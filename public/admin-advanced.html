Step 1 — Create a Stripe Account

  1. Go to stripe.com and sign up or log in.
  2. Complete your account setup (business details, bank account for payouts).
  3. On the Stripe Dashboard, toggle Test mode on while developing, then switch to Live mode when ready.

Step 2 — Get your API Keys

  1. In the Stripe Dashboard, go to Developers → API Keys.
  2. Copy your Publishable key (starts with pk_test_ or pk_live_).
  3. Copy your Secret key (starts with sk_test_ or sk_live_). Keep this private — never expose it in frontend code.
  4. Add these to your server's .env file:
    STRIPE_PUBLISHABLE_KEY=pk_live_...
STRIPE_SECRET_KEY=sk_live_...

Step 3 — Create Products & Prices

  1. In Stripe go to Product Catalog → Add product.
  2. Create one product per tip tier (e.g. "Supporter $5", "Fan $10", "Superfan $20"). Set pricing as One time.
  3. After saving, copy the Product ID (starts with prod_) and Price ID (starts with price_) for each tier.
  4. Add them to .env as comma-separated lists in the same order:
    STRIPE_TIER_PRODUCT_IDS=prod_aaa,prod_bbb,prod_ccc
STRIPE_TIER_PRICE_IDS=price_aaa,price_bbb,price_ccc

Step 4 — Configure the Webhook

Stripe sends a webhook event when a payment completes. This is what marks a transaction as Paid in the admin panel.

  1. In Stripe go to Developers → Webhooks → Add endpoint.
  2. Set the endpoint URL to:
    https://yourdomain.com/api/stripe-webhook.php
  3. Under Select events, add:
    • checkout.session.completed
    • checkout.session.expired
    • charge.refunded
  4. Click Add endpoint. Then reveal the Signing secret (starts with whsec_) and set it in .env:
    STRIPE_WEBHOOK_SECRET=whsec_...
  5. Important: Without STRIPE_WEBHOOK_SECRET, all webhook requests will be rejected and transactions will stay in Pending status.

Step 5 — Set Active Payment Processor

In your .env file ensure:

PAYMENT_PROCESSOR=stripe

You can also change this in the Settings → Payment Processors panel below.

Step 6 — Full .env Example

PAYMENT_PROCESSOR=stripe
STRIPE_SECRET_KEY=sk_live_...
STRIPE_PUBLISHABLE_KEY=pk_live_...
STRIPE_WEBHOOK_SECRET=whsec_...
STRIPE_TIER_PRODUCT_IDS=prod_aaa,prod_bbb,prod_ccc
STRIPE_TIER_PRICE_IDS=price_aaa,price_bbb,price_ccc
ADMIN_DASHBOARD_TOKEN=change-this-to-a-strong-secret
BASE_URL=https://yourdomain.com

Troubleshooting Transactions

  • Transaction stays Pending: The Stripe webhook has not fired or the STRIPE_WEBHOOK_SECRET is wrong. Check the Webhooks tab for events.
  • No transactions appear at all: Confirm STRIPE_TIER_PRICE_IDS match the prices in your Stripe Dashboard exactly.
  • Webhook signature invalid errors: Make sure you copied the signing secret from the correct endpoint in Stripe (test vs live mode).
  • Testing locally: Use the Stripe CLI: stripe listen --forward-to localhost/api/stripe-webhook.php — this gives you a temporary webhook secret for local testing.
  • After going live: Rotate all test keys to live keys and update the webhook endpoint URL and signing secret.
